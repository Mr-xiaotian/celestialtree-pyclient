为 src/ 下的 Python 源文件生成对应的 docs/ 文档。

## 规则

1. 扫描 `src/` 目录下所有 `.py` 文件
2. 对每个 `.py` 文件，在 `docs/` 下建立保持相对路径一一对应的 `.md` 文件
   - 例如 `src/celestialtree/tools/formatters.py` → `docs/tools/formatters.md`
   - `src/<package>/` 前缀去掉顶层包名，直接映射到 `docs/` 下
3. 如果对应的 `.md` 已存在且内容与源文件一致（源文件未变更），跳过不覆盖
4. 如果对应的 `.md` 已存在但源文件有变更，更新文档内容

## 文档内容要求

每个 `.md` 文件应包含：

- **标题**: 使用文件相对路径作为一级标题（如 `# tools/formatters.py`）
- **模块说明**: 一句话描述模块用途
- **类**: 列出所有类，包含：
  - 构造函数签名和参数表格（参数 | 类型 | 默认值 | 说明）
  - 所有公开方法的签名、参数说明、返回值
- **函数**: 列出所有模块级函数，包含签名、参数表格、返回值
- **使用示例**: 如果是用户直接使用的模块，附上简短代码示例
- **特殊说明**: 如自动生成的文件（protobuf/grpc），标注"请勿手动编辑"

## 文档风格

- 使用中文撰写
- 参数表格使用 Markdown 表格格式
- 代码块标注语言（python / bash）
- 保持简洁，不重复 docstring 中已有的内容，侧重 API 参考

## 参数

如果用户指定了 `$ARGUMENTS`，则只为匹配的文件生成文档（支持 glob 模式）。否则为所有 `.py` 文件生成。
